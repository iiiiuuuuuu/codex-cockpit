访问控制

入口 apikey 保护本地代理入口；为空时，请求不会校验。

入口 apikey

入口 apikey 保护 Airouter 本地代理入口，不是 OpenAI、Codex 或 Claude 的上游 API Key，也不会转发给上游。
客户端请求 /v1/* 时，带上任意一个已配置的 key 即可；可以用 Authorization: Bearer <apikey>，也可以用 x-api-key: <apikey>。
配置多个 key 时，任意一个匹配都能访问，适合不同设备或临时授权共用；删除某个 key 只会影响使用它的客户端。列表为空则不校验。

代理入口

配置 Airouter 对外服务端口，以及访问上游时借用的本机 VPN/代理软件端口。

端口配置

Airouter 在你本机监听的对外服务端口。Codex、OpenAI SDK 或其他客户端会连到这个端口，例如把 `http://localhost:3009/v1` 填到 Base URL。

这里填写你本地 VPN/代理软件正在监听的端口，例如 Clash、Surge、V2RayN 常见的 7890。填了以后，Airouter 访问上游时会通过这个代理出网；留空则直连。

代理访问地址
http://localhost:3009/v1
把这个地址填到 Codex/OpenAI 客户端的 Base URL；Responses 接口为 http://localhost:3009/v1/responses。
